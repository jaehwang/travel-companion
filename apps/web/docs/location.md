# 위치 정보 저장 규칙

## 위치 출처

체크인의 위치 정보는 세 가지 출처에서 올 수 있다.

| 출처 | 설명 |
|------|------|
| 수동 선택 | 사용자가 장소 검색 또는 지도에서 직접 위치를 선택 |
| 사진 GPS | 첨부한 사진의 EXIF 데이터에서 추출한 GPS 좌표 |
| 현재 위치 | 체크인 시점의 기기 위치 (Geolocation API) |

## 우선순위

**수동 선택 > 사진 GPS > 현재 위치**

핵심 원칙: 명시적 행동이 암묵적 행동보다 우선하고, 나중에 한 명시적 선택이 이전 것을 덮어쓴다.

## 케이스별 동작

### 사진 첨부 시
- 현재 위치만 있을 때 → 사진 GPS로 자동 교체
- 수동 선택이 이미 있을 때 → 수동 선택 유지 (사진 GPS 무시)

### 사진 제거 시
- 수동 선택으로 설정된 위치 → 그대로 유지
- 사진 GPS로 설정된 위치 → 위치도 함께 초기화

### 수동 선택 후 사진 첨부
- 수동 선택 유지 (사진 GPS 자동 적용 안 함)

### 위치 직접 삭제 시
- 출처 무관하게 위치 초기화

## 구현

`CheckinForm.tsx`에서 `locationSourceRef`(useRef)로 위치 출처를 추적한다.

```
'manual' | 'photo' | null
```

- `'manual'`: 장소 검색 또는 지도에서 직접 선택
- `'photo'`: 사진 GPS에서 자동 추출
- `null`: 위치 없음

ref를 사용하는 이유: `onGpsExtracted` 콜백은 비동기로 호출되므로 클로저에서 최신 값을 읽기 위해 state 대신 ref 사용.
