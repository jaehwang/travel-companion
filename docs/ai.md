# AI 기능

## 여행 태그라인 생성 (Trip Tagline)

선택된 여행의 분위기를 한 줄 문구로 표현해 주는 기능이다.
여행 제목, 설명, 장소, 일정, 체크인 내용을 바탕으로 Gemini가 재치 있는 한국어 문구를 생성한다.

### UI

- 체크인 페이지(`/checkin`)에서 여행을 선택하면 여행 정보 카드 아래에 자동 표시된다.
- 로딩 중에는 "두근, 두근..." 메시지를 보여 준다.
- 새로고침 버튼으로 문구를 다시 생성할 수 있다.
- 여행 정보가 바뀌면(제목·설명·장소·일정) 자동으로 재생성된다.
- 같은 여행을 다시 선택하면 캐시된 문구를 즉시 표시한다(세션 유지).

### 파일 구조

| 파일 | 역할 |
|---|---|
| `app/api/trips/[id]/tagline/route.ts` | POST 엔드포인트. Supabase에서 여행·체크인 정보를 읽어 Gemini 호출 |
| `lib/ai/tripTagline.ts` | 프롬프트 빌드(`buildTripTaglinePrompt`) 및 응답 정규화(`normalizeTripTagline`) |
| `app/checkin/hooks/useTripTagline.ts` | 클라이언트 훅. 캐싱, 로딩 상태, 새로고침 로직 관리 |

### API

```
POST /api/trips/:id/tagline
```

- 인증 필요 (Supabase Auth)
- 응답: `{ tagline: string }` (32자 이내)
- Gemini API 미설정 시 503 반환

### Gemini 설정

- 모델: `gemini-2.5-flash-lite`
- 환경 변수: `GEMINI_API_KEY`
- 패키지: `@google/genai`

### 프롬프트 설계 (`lib/ai/tripTagline.ts`)

목표 톤: "피식 웃기고, 여행 분위기가 느껴지고, 너무 오버하지 않음"

프롬프트에 포함되는 정보:
- 여행 제목, 설명, 대표 장소, 시작일, 종료일, 체크인 개수
- 체크인 최대 6개의 요약 (날짜 | 장소 | 제목 | 메모 | 카테고리)

출력 조건:
- 한 줄, 32자 이내
- 이모지·따옴표·해시태그·줄바꿈 금지
- 제목을 그대로 반복하지 않음
- 말장난, 반전, 상황 묘사를 가볍게 섞을 수 있음

### 캐싱 전략 (`useTripTagline.ts`)

- 캐시 키: 여행 ID
- 캐시 유효성: 여행의 제목·설명·장소·시작일·종료일로 만든 시그니처로 판단
- 시그니처가 바뀌면 자동 재생성
- 새로고침 버튼은 캐시 엔트리를 삭제하고 강제 재요청
- 캐시는 컴포넌트 상태(메모리)에만 유지 → 페이지 새로고침 시 초기화
